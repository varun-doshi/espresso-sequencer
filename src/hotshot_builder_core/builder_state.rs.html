197/// [`best_builder_states_to_extend`] is a utility function that is used to
198/// in order to determine which [`BuilderState`]s are the best fit to extend
199/// from.
200///
201/// This function is designed to inspect the current state of the global state
202/// in order to determine which [`BuilderState`]s are the best fit to extend
203/// from. We only want to use information from [`GlobalState`] as otherwise
204/// we would have some insider knowledge unique to our specific [`BuilderState`]
205/// rather than knowledge that is available to all [`BuilderState`]s. In fact,
206/// in order to ensure this, this function lives outside of the [`BuilderState`]
207/// itself.
208///
209/// In an ideal circumstance the best [`BuilderState`] to extend from is going to
210/// be the one that is immediately preceding the [`QuorumProposalWrapper`] that we are
211/// attempting to extend from. However, if all we know is the view number of
212/// the [`QuorumProposalWrapper`] that we are attempting to extend from, then we may end
213/// up in a scenario where we have multiple [`BuilderState`]s that are all equally
214/// valid to extend from.  When this happens, we have the potential for a data
215/// race.
216///
217/// The primary cause of this has to due with the interface of the
218/// [`ProxyGlobalState`](crate::service::ProxyGlobalState)'s API.  In general,
219/// we want to be able to retrieve a [`BuilderState`] via the [`BuilderStateId`].
220/// The [`BuilderStateId`] only references a [`ViewNumber`](hotshot_types::data::ViewNumber)
221/// and a [`VidCommitment`] While this information is available in the [`QuorumProposalWrapper`],
222/// it only helps us to rule out [`BuilderState`]s that already exist.
223/// It does **NOT** help us to pick a [`BuilderState`] that is the best fit to extend from.
224///
225/// This is where the `justify_qc` comes in to consideration.  The `justify_qc`
226/// contains the previous [`ViewNumber`](hotshot_types::data::ViewNumber) that is
227/// being extended from, and in addition it also contains the previous [`Commitment<Leaf<Types>>`]
228/// that is being built on top of.  Since our [`BuilderState`]s store identifying
229/// information that contains this same `leaf_commit` we can compare these
230/// directly to ensure that we are extending from the correct [`BuilderState`].
231///
232/// This function determines the best [`BuilderState`] in the following steps:
233///
234/// 1. If we have a [`BuilderState`] that is already spawned for the current
235///    [`QuorumProposalWrapper`], then we should should return no states, as one already
236///    exists.  This will prevent us from attempting to spawn duplicate
237///    [`BuilderState`]s.
238/// 2. Attempt to find all [`BuilderState`]s that are recorded within
239///    [`GlobalState`] that have matching view number and leaf commitments. There
240///    *should* only be one of these.  But all would be valid extension points.
241/// 3. If we can't find any [`BuilderState`]s that match the view number
242///    and leaf commitment, then we should return for the maximum stored view
243///    number that is smaller than the current [`QuorumProposalWrapper`].
244/// 4. If there is is only one [`BuilderState`] stored in the [`GlobalState`], then
245///    we should return that [`BuilderState`] as the best fit.
246/// 5. If none of the other criteria match, we return an empty result as it is
247///    unclear what to do in this case.
248///
249/// > Note: Any time this function returns more than a single entry in its
250/// > [HashSet] result, there is a potential for a race condition.  This is
251/// > because there are multiple [BuilderState]s that are equally valid to
252/// > extend from.  This race could be avoided by just picking one of the
253/// > entries in the resulting [HashSet], but this is not done here in order
254/// > to allow us to highlight the possibility of the race.
255async fn best_builder_states_to_extend<Types: NodeType>(
256    quorum_proposal: Arc<Proposal<Types, QuorumProposalWrapper<Types>>>,
257    global_state: Arc<RwLock<GlobalState<Types>>>,
258) -> HashSet<BuilderStateId<Types>> {
259    let current_view_number = quorum_proposal.data.view_number();
260    let current_commitment = quorum_proposal.data.block_header().payload_commitment();
261    let current_builder_state_id = BuilderStateId::<Types> {
262        parent_commitment: current_commitment,
263        parent_view: current_view_number,
264    };
265
266    let global_state_read_lock = global_state.read_arc().await;
267
268    // The first step is to check if we already have a spawned [BuilderState].
269    // If we do, then we should indicate that there is no best fit, as we
270    // don't want to spawn another [BuilderState].
271    if global_state_read_lock
272        .spawned_builder_states
273        .contains_key(&current_builder_state_id)
274    {
275        // We already have a spawned [BuilderState] for this proposal.
276        // So we should just ignore it.
277        return HashSet::new();
278    }
279
280    // Next we want to see if there is an immediate match for a [BuilderState]
281    // that we can extend from.  This is the most ideal situation, as it
282    // implies that we are extending from the correct [BuilderState].
283    // We do this by checking the `justify_qc` stored within the
284    // [QuorumProposal], and checking it against the current spawned
285    // [BuilderState]s
286    let justify_qc = quorum_proposal.data.justify_qc();
287    let existing_states: HashSet<_> = global_state_read_lock
288        .spawned_builder_states
289        .iter()
290        .filter(
291            |(_, (parent_block_references, _))| match parent_block_references {
292                None => false,
293                Some(parent_block_references) => {
294                    parent_block_references.leaf_commit == justify_qc.data.leaf_commit
295                        && parent_block_references.view_number == justify_qc.view_number
296                },
297            },
298        )
299        .map(|(builder_state_id, _)| builder_state_id.clone())
300        .collect();
301
302    // If we found any matching [BuilderState]s, then we should return them
303    // as the best fit.
304    if !existing_states.is_empty() {
305        return existing_states;
306    }
307
308    // At this point, we don't have any "ideal" matches or scenarios.  So we
309    // need to look for a suitable fall-back. The best fallback condition to
310    // start with is any [BuilderState] that has the maximum spawned view
311    // number whose value is smaller than the current [QuorumProposal].
312    let maximum_stored_view_number_smaller_than_quorum_proposal = global_state_read_lock
313        .spawned_builder_states
314        .keys()
315        .map(|builder_state_id| builder_state_id.parent_view)
316        .filter(|view_number| *view_number < current_view_number)
317        .max();
318
319    // If we have a maximum view number that meets our criteria, then we should
320    // return all [BuilderStateId]s that match this view number.
321    // This can lead to multiple [BuilderStateId]s being returned.
322    if let Some(maximum_stored_view_number_smaller_than_quorum_proposal) =
323        maximum_stored_view_number_smaller_than_quorum_proposal
324    {
325        // If we are the maximum stored view number smaller than the quorum
326        // proposal's view number, then we are the best fit.
327        let mut result = HashSet::new();
328        for builder_state_id in
329            global_state_read_lock
330                .spawned_builder_states
331                .keys()
332                .filter(|builder_state_id| {
333                    builder_state_id.parent_view
334                        == maximum_stored_view_number_smaller_than_quorum_proposal
335                })
336        {
337            result.insert(builder_state_id.clone());
338        }
339        return result;
340    }
341
342    // This is our last ditch effort to continue making progress.  If there is
343    // only one [BuilderState] active, then we should return that as the best
344    // fit, as it will be the only way we can continue making progress with
345    // the builder.
346    if global_state_read_lock.spawned_builder_states.len() == 1 {
347        let mut result = HashSet::new();
348        for builder_state_id in global_state_read_lock.spawned_builder_states.keys() {
349            result.insert(builder_state_id.clone());
350        }
351        return result;
352    }
353
354    // This implies that there are only larger [BuilderState]s active than
355    // the one we are.  This is weird, it implies that some sort of time
356    // travel has occurred view-wise.  It is unclear what to do in this
357    // situation.
358
359    HashSet::new()
360}

362impl<Types: NodeType, V: Versions> BuilderState<Types, V> {
363    /// Utility method that attempts to determine whether
364    /// we are among the best [`BuilderState`]s to extend from.
365    async fn am_i_the_best_builder_state_to_extend(
366        &self,
367        quorum_proposal: Arc<Proposal<Types, QuorumProposalWrapper<Types>>>,
368    ) -> bool {
369        let best_builder_states_to_extend =
370            best_builder_states_to_extend(quorum_proposal.clone(), self.global_state.clone()).await;
371
372        tracing::debug!(
373            "{}@{} thinks these are the best builder states to extend from: {:?} for proposal {}@{}",
374            self.parent_block_references.vid_commitment,
375            self.parent_block_references.view_number.u64(),
376            best_builder_states_to_extend
377                .iter()
378                .map(|builder_state_id| format!(
379                    "{}@{}",
380                    builder_state_id.parent_commitment,
381                    builder_state_id.parent_view.u64()
382                ))
383                .collect::<Vec<String>>(),
384            quorum_proposal.data.block_header().payload_commitment(),
385            quorum_proposal.data.view_number().u64(),
386        );
387
388        // We are a best fit if we are contained within the returned set of
389        // best [BuilderState]s to extend from.
390        best_builder_states_to_extend.contains(&BuilderStateId {
391            parent_commitment: self.parent_block_references.vid_commitment,
392            parent_view: self.parent_block_references.view_number,
393        })
394    }

396    /// processing the DA proposal
397    #[tracing::instrument(skip_all, name = "process da proposal",
398                                    fields(builder_parent_block_references = %self.parent_block_references))]
399    async fn process_da_proposal(&mut self, da_msg: DaProposalMessage<Types>) {
400        tracing::debug!(
401            "Builder Received DA message for view {:?}",
402            da_msg.proposal.data.view_number
403        );
404
405        // we do not have the option to ignore DA proposals if we want to be able to handle failed view reorgs.
406
407        // If the respective builder state exists to handle the request
408        let proposal = da_msg.proposal.clone();
409
410        // get the view number and encoded txns from the da_proposal_data
411        let view_number = proposal.data.view_number;
412        let encoded_txns = &proposal.data.encoded_transactions;
413
414        let metadata = &proposal.data.metadata;
415
416        // form a block payload from the encoded transactions
417        let block_payload =
418            <Types::BlockPayload as BlockPayload<Types>>::from_bytes(encoded_txns, metadata);
419        // get the builder commitment from the block payload
420        let payload_builder_commitment = block_payload.builder_commitment(metadata);
421
422        tracing::debug!(
423            "Extracted builder commitment from the da proposal: {:?}",
424            payload_builder_commitment
425        );
426
427        // form the DA proposal info
428        let da_proposal_info = DAProposalInfo {
429            view_number,
430            proposal,
431        };
432
433        let std::collections::hash_map::Entry::Vacant(e) = self
434            .da_proposal_payload_commit_to_da_proposal
435            .entry((payload_builder_commitment.clone(), view_number))
436        else {
437            tracing::debug!("Payload commitment already exists in the da_proposal_payload_commit_to_da_proposal hashmap, so ignoring it");
438            return;
439        };
440
441        // if we have matching da and quorum proposals, we can skip storing the one, and remove
442        // the other from storage, and call build_block with both, to save a little space.
443
444        let Entry::Occupied(quorum_proposal) = self
445            .quorum_proposal_payload_commit_to_quorum_proposal
446            .entry((payload_builder_commitment.clone(), view_number))
447        else {
448            e.insert(da_proposal_info);
449            return;
450        };
451
452        let quorum_proposal = quorum_proposal.remove();
453
454        // if we have a matching quorum proposal
455        //  if (this is the correct parent or
456        //      (the correct parent is missing and this is the highest view))
457        //    spawn a clone
458        if quorum_proposal.data.view_number() != view_number {
459            tracing::debug!("Not spawning a clone despite matching DA and quorum payload commitments, as they corresponds to different view numbers");
460            return;
461        }
462
463        tracing::info!(
464            "Spawning a clone from process DA proposal for view number: {:?}",
465            view_number
466        );
467        // remove this entry from quorum_proposal_payload_commit_to_quorum_proposal
468        self.quorum_proposal_payload_commit_to_quorum_proposal
469            .remove(&(payload_builder_commitment.clone(), view_number));
470        self.spawn_clone_that_extends_self(da_proposal_info, quorum_proposal.clone())
471            .await;
472    }
473
474    /// processing the quorum proposal
475    //#[tracing::instrument(skip_all, name = "Process Quorum Proposal")]
476    #[tracing::instrument(skip_all, name = "process quorum proposal",
477                                    fields(builder_parent_block_references = %self.parent_block_references))]
478    async fn process_quorum_proposal(&mut self, quorum_msg: QuorumProposalMessage<Types>) {
479        tracing::debug!(
480            "Builder Received Quorum proposal message for view {:?}",
481            quorum_msg.proposal.data.view_number()
482        );
483
484        // Two cases to handle:
485        // Case 1: Bootstrapping phase
486        // Case 2: No intended builder state exist
487        // To handle both cases, we can have the highest view number builder state running
488        // and only doing the insertion if and only if intended builder state for a particulat view is not present
489        // check the presence of quorum_proposal.data.view_number-1 in the spawned_builder_states list
490        let quorum_proposal = &quorum_msg.proposal;
491        let view_number = quorum_proposal.data.view_number();
492        let payload_builder_commitment = quorum_proposal.data.block_header().builder_commitment();
493
494        tracing::debug!(
495            "Extracted payload builder commitment from the quorum proposal: {:?}",
496            payload_builder_commitment
497        );
498
499        let std::collections::hash_map::Entry::Vacant(e) = self
500            .quorum_proposal_payload_commit_to_quorum_proposal
501            .entry((payload_builder_commitment.clone(), view_number))
502        else {
503            tracing::debug!("Payload commitment already exists in the quorum_proposal_payload_commit_to_quorum_proposal hashmap, so ignoring it");
504            return;
505        };
506
507        // first check whether vid_commitment exists in the quorum_proposal_payload_commit_to_quorum_proposal hashmap, if yer, ignore it, otherwise validate it and later insert in
508        // if we have matching da and quorum proposals, we can skip storing the one, and remove the other from storage, and call build_block with both, to save a little space.
509        let Entry::Occupied(da_proposal) = self
510            .da_proposal_payload_commit_to_da_proposal
511            .entry((payload_builder_commitment.clone(), view_number))
512        else {
513            e.insert(quorum_proposal.clone());
514            return;
515        };
516
517        let da_proposal_info = da_proposal.remove();
518        // remove the entry from the da_proposal_payload_commit_to_da_proposal hashmap
519        self.da_proposal_payload_commit_to_da_proposal
520            .remove(&(payload_builder_commitment.clone(), view_number));
521
522        // also make sure we clone for the same view number( check incase payload commitments are same)
523        if da_proposal_info.view_number != view_number {
524            tracing::debug!("Not spawning a clone despite matching DA and quorum payload commitments, as they corresponds to different view numbers");
525            return;
526        }
527
528        tracing::info!(
529            "Spawning a clone from process quorum proposal for view number: {:?}",
530            view_number
531        );
532
533        self.spawn_clone_that_extends_self(da_proposal_info, quorum_proposal.clone())
534            .await;
535    }

537    /// A helper function that is used by both [`BuilderState::process_da_proposal`]
538    /// and [`BuilderState::process_quorum_proposal`] to spawn a new [`BuilderState`]
539    /// that extends from the current [`BuilderState`].
540    ///
541    /// This helper function also adds additional checks in order to ensure
542    /// that the [`BuilderState`] that is being spawned is the best fit for the
543    /// [`QuorumProposalWrapper`] that is being extended from.
544    async fn spawn_clone_that_extends_self(
545        &mut self,
546        da_proposal_info: DAProposalInfo<Types>,
547        quorum_proposal: Arc<Proposal<Types, QuorumProposalWrapper<Types>>>,
548    ) {
549        if !self
550            .am_i_the_best_builder_state_to_extend(quorum_proposal.clone())
551            .await
552        {
553            tracing::debug!(
554                "{} is not the best fit for forking, {}@{}, so ignoring the quorum proposal, and leaving it to another BuilderState",
555                self.parent_block_references,
556                quorum_proposal.data.block_header().payload_commitment(),
557                quorum_proposal.data.view_number().u64(),
558            );
559            return;
560        }
561
562        let (req_sender, req_receiver) = broadcast(self.req_receiver.capacity());
563
564        tracing::debug!(
565            "extending BuilderState with a clone from {} with new proposal {}@{}",
566            self.parent_block_references,
567            quorum_proposal.data.block_header().payload_commitment(),
568            quorum_proposal.data.view_number().u64()
569        );
570        // We literally fork ourselves
571        self.clone_with_receiver(req_receiver)
572            .spawn_clone(da_proposal_info, quorum_proposal.clone(), req_sender)
573            .await;
574    }

577    #[tracing::instrument(skip_all, name = "process decide event",
578                                   fields(builder_parent_block_references = %self.parent_block_references))]
579    async fn process_decide_event(&mut self, decide_msg: DecideMessage<Types>) -> Option<Status> {
580        // Exit out all the builder states if their parent_block_references.view_number is less than the latest_decide_view_number
581        // The only exception is that we want to keep the highest view number builder state active to ensure that
582        // we have a builder state to handle the incoming DA and quorum proposals
583        let decide_view_number = decide_msg.latest_decide_view_number;
584
585        let retained_view_cutoff = self
586            .global_state
587            .write_arc()
588            .await
589            .remove_handles(decide_view_number);
590        if self.parent_block_references.view_number < retained_view_cutoff {
591            tracing::info!(
592                "Decide@{:?}; Task@{:?} exiting; views < {:?} being reclaimed",
593                decide_view_number.u64(),
594                self.parent_block_references.view_number.u64(),
595                retained_view_cutoff.u64(),
596            );
597            return Some(Status::ShouldExit);
598        }
599        tracing::info!(
600            "Decide@{:?}; Task@{:?} not exiting; views >= {:?} being retained",
601            decide_view_number.u64(),
602            self.parent_block_references.view_number.u64(),
603            retained_view_cutoff.u64(),
604        );
605
606        Some(Status::ShouldContinue)
607    }

609    // spawn a clone of the builder state
610    #[tracing::instrument(skip_all, name = "spawn_clone",
611                                    fields(builder_parent_block_references = %self.parent_block_references))]
612    async fn spawn_clone(
613        mut self,
614        da_proposal_info: DAProposalInfo<Types>,
615        quorum_proposal: Arc<Proposal<Types, QuorumProposalWrapper<Types>>>,
616        req_sender: BroadcastSender<MessageType<Types>>,
617    ) {
618        let leaf = Leaf2::from_quorum_proposal(&quorum_proposal.data);
619
620        // We replace our parent_block_references with information from the
621        // quorum proposal.  This is identifying the block that this specific
622        // instance of [BuilderState] is attempting to build for.
623        self.parent_block_references = ParentBlockReferences {
624            view_number: quorum_proposal.data.view_number(),
625            vid_commitment: quorum_proposal.data.block_header().payload_commitment(),
626            leaf_commit: leaf.commit(),
627            builder_commitment: quorum_proposal.data.block_header().builder_commitment(),
628            // Unused in old legacy builder:
629            last_nonempty_view: None,
630            tx_count: 0,
631        };
632
633        let builder_state_id = BuilderStateId {
634            parent_commitment: self.parent_block_references.vid_commitment,
635            parent_view: self.parent_block_references.view_number,
636        };
637
638        {
639            // Let's ensure that we don't already have one of these BuilderStates
640            // running already.
641
642            let global_state_read_lock = self.global_state.read_arc().await;
643            if global_state_read_lock
644                .spawned_builder_states
645                .contains_key(&builder_state_id)
646            {
647                tracing::warn!(
648                    "Aborting spawn_clone, builder state already exists in spawned_builder_states: {:?}",
649                    builder_state_id
650                );
651                return;
652            }
653        }
654
655        let encoded_txns = &da_proposal_info.proposal.data.encoded_transactions;
656        let metadata = &da_proposal_info.proposal.data.metadata;
657
658        let block_payload =
659            <Types::BlockPayload as BlockPayload<Types>>::from_bytes(encoded_txns, metadata);
660        let txn_commitments = block_payload.transaction_commitments(metadata);
661
662        for tx in txn_commitments.iter() {
663            self.txns_in_queue.remove(tx);
664        }
665
666        self.included_txns.extend(txn_commitments.iter());
667        self.tx_queue
668            .retain(|tx| self.txns_in_queue.contains(&tx.commit));
669
670        if !txn_commitments.is_empty() {
671            self.allow_empty_block_until = Some(Types::View::new(
672                da_proposal_info.view_number.u64() + ALLOW_EMPTY_BLOCK_PERIOD,
673            ));
674        }
675
676        // register the spawned builder state to spawned_builder_states in the global state
677        self.global_state.write_arc().await.register_builder_state(
678            BuilderStateId {
679                parent_commitment: self.parent_block_references.vid_commitment,
680                parent_view: self.parent_block_references.view_number,
681            },
682            self.parent_block_references.clone(),
683            req_sender,
684        );
685
686        self.event_loop();
687    }

689    // build a block
690    #[tracing::instrument(skip_all, name = "build block",
691                                    fields(builder_parent_block_references = %self.parent_block_references))]
692    async fn build_block(
693        &mut self,
694        state_id: BuilderStateId<Types>,
695    ) -> Option<BuildBlockInfo<Types>> {
696        let timeout_after = Instant::now() + self.maximize_txn_capture_timeout;
697        let sleep_interval = self.maximize_txn_capture_timeout / 10;
698        while Instant::now() <= timeout_after {
699            self.collect_txns(timeout_after).await;
700
701            if !self.tx_queue.is_empty() // we have transactions
702            || Instant::now() + sleep_interval > timeout_after
703            // we don't have time for another iteration
704            {
705                break;
706            }
707
708            sleep(sleep_interval).await
709        }
710
711        // should_prioritize_finalization is a flag that is used to determine
712        // whether we should return empty blocks or not.
713
714        let should_prioritize_finalization = self
715            .allow_empty_block_until
716            .map(|until| state_id.parent_view < until)
717            .unwrap_or(false);
718
719        if self.tx_queue.is_empty() && !should_prioritize_finalization {
720            // Don't build an empty block
721            return None;
722        }
723
724        let max_block_size = self
725            .global_state
726            .read_arc()
727            .await
728            .block_size_limits
729            .max_block_size;
730        let transactions_to_include = self.tx_queue.iter().scan(0, |total_size, tx| {
731            let prev_size = *total_size;
732            *total_size += tx.len;
733            // We will include one transaction over our target block length
734            // if it's the first transaction in queue, otherwise we'd have a possible failure
735            // state where a single transaction larger than target block state is stuck in
736            // queue and we just build empty blocks forever
737            if *total_size >= max_block_size && prev_size != 0 {
738                None
739            } else {
740                Some(tx.tx.clone())
741            }
742        });
743
744        let Ok((payload, metadata)) =
745            <Types::BlockPayload as BlockPayload<Types>>::from_transactions(
746                transactions_to_include,
747                &self.validated_state,
748                &self.instance_state,
749            )
750            .await
751        else {
752            tracing::warn!("build block, returning None");
753            return None;
754        };
755
756        let builder_hash = payload.builder_commitment(&metadata);
757        // count the number of txns
758        let actual_txn_count = payload.num_transactions(&metadata);
759
760        // Payload is empty despite us checking that tx_queue isn't empty earlier.
761        //
762        // This means that the block was truncated due to *sequencer* block length
763        // limits, which are different from our `max_block_size`. There's no good way
764        // for us to check for this in advance, so we detect transactions too big for
765        // the sequencer indirectly, by observing that we passed some transactions
766        // to `<Types::BlockPayload as BlockPayload<Types>>::from_transactions`, but
767        // it returned an empty block.
768        // Thus we deduce that the first transaction in our queue is too big to *ever*
769        // be included, because it alone goes over sequencer's block size limit.
770        // We need to drop it and mark as "included" so that if we receive
771        // it again we don't even bother with it.
772        if actual_txn_count == 0 && !should_prioritize_finalization {
773            if let Some(txn) = self.tx_queue.pop_front() {
774                self.txns_in_queue.remove(&txn.commit);
775                self.included_txns.insert(txn.commit);
776            };
777            return None;
778        }
779
780        // insert the recently built block into the builder commitments
781        self.builder_commitments
782            .insert((state_id, builder_hash.clone()));
783
784        let encoded_txns: Vec<u8> = payload.encode().to_vec();
785        let block_size: u64 = encoded_txns.len() as u64;
786        let offered_fee: u64 = self.base_fee * block_size;
787
788        // TODO Add precompute back.
789        let (trigger_send, _) = oneshot::channel();
790
791        tracing::info!(
792            "Builder view num {:?}, building block with {:?} txns, with builder hash {:?}",
793            self.parent_block_references.view_number,
794            actual_txn_count,
795            builder_hash
796        );
797
798        Some(BuildBlockInfo {
799            id: BlockId {
800                view: self.parent_block_references.view_number,
801                hash: builder_hash,
802            },
803            block_size,
804            offered_fee,
805            block_payload: payload,
806            metadata,
807            vid_trigger: trigger_send,
808            truncated: actual_txn_count < self.tx_queue.len(),
809        })
810    }

812    async fn process_block_request(&mut self, req: RequestMessage<Types>) {
813        // If a spawned clone is active then it will handle the request, otherwise the highest view num builder will handle
814        if req.state_id.parent_commitment != self.parent_block_references.vid_commitment
815            || req.state_id.parent_view != self.parent_block_references.view_number
816        {
817            tracing::debug!(
818                "Builder {:?} Requested Builder commitment does not match the built_from_view, so ignoring it",
819                self.parent_block_references.view_number
820            );
821            return;
822        }
823
824        let highest_view_num_builder_id = self
825            .global_state
826            .read_arc()
827            .await
828            .highest_view_num_builder_id
829            .clone();
830
831        if highest_view_num_builder_id.parent_view != self.parent_block_references.view_number {
832            tracing::debug!(
833                "Builder {:?} Requested Builder commitment does not match the highest_view_num_builder_id, so ignoring it",
834                self.parent_block_references.view_number
835            );
836            return;
837        }
838
839        tracing::info!(
840            "Request for parent {} handled by builder with view {:?}",
841            req.state_id,
842            self.parent_block_references.view_number,
843        );
844        let response = self.build_block(req.state_id.clone()).await;
845
846        let Some(response) = response else {
847            tracing::debug!("No response to send");
848            return;
849        };
850
851        // form the response message
852        let response_msg = ResponseMessage {
853            builder_hash: response.id.hash.clone(),
854            block_size: response.block_size,
855            offered_fee: response.offered_fee,
856        };
857
858        let builder_hash = response.id.hash.clone();
859        self.global_state.write_arc().await.update_global_state(
860            req.state_id.clone(),
861            response,
862            response_msg.clone(),
863        );
864
865        // ... and finally, send the response
866        if let Err(e) = req.response_channel.send(response_msg) {
867            tracing::warn!(
868                "Builder {:?} failed to send response to {:?} with builder hash {:?}, Err: {:?}",
869                self.parent_block_references.view_number,
870                req,
871                builder_hash,
872                e
873            );
874            return;
875        }
876
877        tracing::info!(
878            "Builder {:?} Sent response to the request{:?} with builder hash {:?}",
879            self.parent_block_references.view_number,
880            req,
881            builder_hash
882        );
883    }

885    #[tracing::instrument(skip_all, name = "event loop",
886                                    fields(builder_parent_block_references = %self.parent_block_references))]
887    pub fn event_loop(mut self) {
888        let _builder_handle = spawn(async move {
889            loop {
890                tracing::debug!(
891                    "Builder {:?} event loop",
892                    self.parent_block_references.view_number
893                );
894                futures::select! {
895                    req = self.req_receiver.next() => {
896                        tracing::debug!("Received request msg in builder {:?}: {:?}", self.parent_block_references.view_number, req);
897                        match req {
898                            Some(req) => {
899                                if let MessageType::RequestMessage(req) = req {
900                                    tracing::debug!(
901                                        "Received request msg in builder {:?}: {:?}",
902                                        self.parent_block_references.view_number,
903                                        req
904                                    );
905                                    self.process_block_request(req).await;
906                                } else {
907                                    tracing::warn!("Unexpected message on requests channel: {:?}", req);
908                                }
909                            }
910                            None => {
911                                tracing::warn!("No more request messages to consume");
912                            }
913                        }
914                    },
915                    da = self.da_proposal_receiver.next() => {
916                        match da {
917                            Some(da) => {
918                                if let MessageType::DaProposalMessage(rda_msg) = da {
919                                    tracing::debug!("Received da proposal msg in builder {:?}:\n {:?}", self.parent_block_references, rda_msg.proposal.data.view_number);
920                                    self.process_da_proposal(rda_msg).await;
921                                } else {
922                                    tracing::warn!("Unexpected message on da proposals channel: {:?}", da);
923                                }
924                            }
925                            None => {
926                                tracing::warn!("No more da proposal messages to consume");
927                            }
928                        }
929                    },
930                    quorum = self.quorum_proposal_receiver.next() => {
931                        match quorum {
932                            Some(quorum) => {
933                                if let MessageType::QuorumProposalMessage(rquorum_msg) = quorum {
934                                    tracing::debug!("Received quorum proposal msg in builder {:?}:\n {:?} for view ", self.parent_block_references, rquorum_msg.proposal.data.view_number());
935                                    self.process_quorum_proposal(rquorum_msg).await;
936                                } else {
937                                    tracing::warn!("Unexpected message on quorum proposals channel: {:?}", quorum);
938                                }
939                            }
940                            None => {
941                                tracing::warn!("No more quorum proposal messages to consume");
942                            }
943                        }
944                    },
945                    decide = self.decide_receiver.next() => {
946                        match decide {
947                            Some(decide) => {
948                                if let MessageType::DecideMessage(rdecide_msg) = decide {
949                                    let latest_decide_view_num = rdecide_msg.latest_decide_view_number;
950                                    tracing::debug!("Received decide msg view {:?} in builder {:?}",
951                                        &latest_decide_view_num,
952                                        self.parent_block_references);
953                                    let decide_status = self.process_decide_event(rdecide_msg).await;
954                                    match decide_status{
955                                        Some(Status::ShouldExit) => {
956                                            tracing::info!("Exiting builder {:?} with decide view {:?}",
957                                                self.parent_block_references,
958                                                &latest_decide_view_num);
959                                            return;
960                                        }
961                                        Some(Status::ShouldContinue) => {
962                                            tracing::debug!("Continuing builder {:?}",
963                                                self.parent_block_references);
964                                            continue;
965                                        }
966                                        None => {
967                                            tracing::warn!("decide_status was None; Continuing builder {:?}",
968                                                self.parent_block_references);
969                                            continue;
970                                        }
971                                    }
972                                } else {
973                                    tracing::warn!("Unexpected message on decide channel: {:?}", decide);
974                                }
975                            }
976                            None => {
977                                tracing::warn!("No more decide messages to consume");
978                            }
979                        }
980                    },
981                };
982            }
983        });
984    }

1089    // collect outstanding transactions
1090    async fn collect_txns(&mut self, timeout_after: Instant) {
1091        while Instant::now() <= timeout_after {
1092            match self.tx_receiver.try_recv() {
1093                Ok(tx) => {
1094                    if self.included_txns.contains(&tx.commit)
1095                        || self.included_txns_old.contains(&tx.commit)
1096                        || self.included_txns_expiring.contains(&tx.commit)
1097                        || self.txns_in_queue.contains(&tx.commit)
1098                    {
1099                        continue;
1100                    }
1101                    self.txns_in_queue.insert(tx.commit);
1102                    self.tx_queue.push_back(tx);
1103                },
1104                Err(async_broadcast::TryRecvError::Empty)
1105                | Err(async_broadcast::TryRecvError::Closed) => {
1106                    break;
1107                },
1108                Err(async_broadcast::TryRecvError::Overflowed(lost)) => {
1109                    tracing::warn!("Missed {lost} transactions due to backlog");
1110                    continue;
1111                },
1112            }
1113        }
1114    }

1375        // Number of views to simulate
1376        const NUM_ROUNDS: usize = 5;

1378        const NUM_TXNS_PER_ROUND: usize = 4;
1379        // Capacity of broadcast channels
1380        const CHANNEL_CAPACITY: usize = NUM_ROUNDS * 5;
1381        // Number of nodes on DA committee

1390            .map(|round| {
1391                (0..NUM_TXNS_PER_ROUND)
1392                    .map(|tx_num| TestTransaction::new(vec![round as u8, tx_num as u8]))
1393                    .collect::<Vec<_>>()
1394            })
1395            .collect::<Vec<_>>();
1396        let mut prev_quorum_proposal: Option<QuorumProposalWrapper<TestTypes>> = None;

1413                .u64_field("view number", leaf.view_number().u64())
1414                .u64_field("block number", leaf.height())
1415                .field("parent Leaf commitment", leaf.parent_commitment())
