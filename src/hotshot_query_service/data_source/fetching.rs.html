13//! Asynchronous retrieval of missing data.
14//!
15//! [`FetchingDataSource`] combines a local storage implementation with a remote data availability
16//! provider to create a data sources which caches data locally, but which is capable of fetching
17//! missing data from a remote source, either proactively or on demand.
18//!
19//! This implementation supports three kinds of data fetching.
20//!
21//! # Proactive Fetching
22//!
23//! Proactive fetching means actively scanning the local database for missing objects and
24//! proactively retrieving them from a remote provider, even if those objects have yet to be
25//! requested by a client. Doing this increases the chance of success and decreases latency when a
26//! client does eventually ask for those objects. This is also the mechanism by which a query
27//! service joining a network late, or having been offline for some time, is able to catch up with
28//! the events on the network that it missed.
29//!
30//! The current implementation of proactive fetching is meant to be the simplest effective algorithm
31//! which still gives us a reasonable range of configuration options for experimentation. It is
32//! subject to change as we learn about the behavior of proactive fetching in a realistic system.
33//!
34//! Proactive fetching is currently implemented by a background task which performs periodic scans
35//! of the database, identifying and retrieving missing objects. This task is generally low
36//! priority, since missing objects are rare, and it will take care not to monopolize resources that
37//! could be used to serve requests. To reduce load and to optimize for the common case where blocks
38//! are usually not missing once they have already been retrieved, we distinguish between _major_
39//! and _minor_ scans.
40//!
41//! Minor scans are lightweight and can run very frequently. They will only look for missing
42//! blocks among blocks that are new since the previous scan. Thus, the more frequently minor
43//! scans run, the less work they have to do. This allows them to run frequently, giving low
44//! latency for retrieval of newly produced blocks that we failed to receive initially. Between
45//! each minor scan, the task will sleep for [a configurable
46//! duration](Builder::with_minor_scan_interval) to wait for new blocks to be produced and give
47//! other tasks full access to all shared resources.
48//!
49//! Every `n`th scan (`n` is [configurable](Builder::with_major_scan_interval)) is a major scan.
50//! These scan all blocks from 0, which guarantees that we will eventually retrieve all blocks, even
51//! if for some reason we have lost a block that we previously had (due to storage failures and
52//! corruptions, or simple bugs in this software). These scans are rather expensive (although they
53//! will release control of shared resources many times during the duration of the scan), but
54//! because it is rather unlikely that a major scan will discover any missing blocks that the next
55//! minor scan would have missed, it is ok if major scans run very infrequently.
56//!
57//! # Active Fetching
58//!
59//! Active fetching means reaching out to a remote data availability provider to retrieve a missing
60//! resource, upon receiving a request for that resource from a client. Not every request for a
61//! missing resource triggers an active fetch. To avoid spamming peers with requests for missing
62//! data, we only actively fetch resources that are known to exist somewhere. This means we can
63//! actively fetch leaves and headers when we are requested a leaf or header by height, whose height
64//! is less than the current chain height. We can fetch a block when the corresponding header exists
65//! (corresponding based on height, hash, or payload hash) or can be actively fetched.
66//!
67//! # Passive Fetching
68//!
69//! For requests that cannot be actively fetched (for example, a block requested by hash, where we
70//! do not have a header proving that a block with that hash exists), we use passive fetching. This
71//! essentially means waiting passively until the query service receives an object that satisfies
72//! the request. This object may be received because it was actively fetched in responsive to a
73//! different request for the same object, one that permitted an active fetch. Or it may have been
74//! fetched [proactively](#proactive-fetching).

174    pub fn new(storage: S, provider: P) -> Self {
175        let mut default_backoff = ExponentialBackoffBuilder::default();
176        default_backoff
177            .with_initial_interval(Duration::from_secs(1))
178            .with_multiplier(2.)
179            .with_max_interval(Duration::from_secs(32))
180            .with_max_elapsed_time(Some(Duration::from_secs(64)));
181
182        Self {
183            storage,
184            provider,
185            backoff: default_backoff,
186            rate_limit: 32,
187            range_chunk_size: 25,
188            // By default, we run minor proactive scans fairly frequently: once every minute. These
189            // scans are cheap (more the more frequently they run) and can help us keep up with
190            // the head of the chain even if our attached consensus instance is behind.
191            minor_scan_interval: Duration::from_secs(60),
192            // Major scans, on the other hand, are rather expensive and not especially important for
193            // usability. We run them rarely, once every 60 minor scans, or once every hour by
194            // default.
195            major_scan_interval: 60,
196            // Major scan offset can be used when starting multiple nodes at the same time, so they
197            // don't all pause for a major scan together.

200            active_fetch_delay: Duration::from_millis(50),
201            chunk_fetch_delay: Duration::from_millis(100),
202            proactive_fetching: true,
203            aggregator: true,
204            aggregator_chunk_size: None,
205            leaf_only: false,
206            _types: Default::default(),
207        }

389/// The most basic kind of data source.
390///
391/// A data source is constructed modularly by combining a [storage](super::storage) implementation
392/// with a [Fetcher](crate::fetching::Fetcher). The former allows the query service to store the
393/// data it has persistently in an easily accessible storage medium, such as the local file system
394/// or a database. This allows it to answer queries efficiently and to maintain its state across
395/// restarts. The latter allows the query service to fetch data that is missing from its storage
396/// from an external data availability provider, such as the Tiramisu DA network or another instance
397/// of the query service.
398///
399/// These two components of a data source are combined in [`FetchingDataSource`], which is the
400/// lowest level kind of data source available. It simply uses the storage implementation to fetch
401/// data when available, and fills in everything else using the fetcher. Various kinds of data
402/// sources can be constructed out of [`FetchingDataSource`] by changing the storage and fetcher
403/// implementations used, and more complex data sources can be built on top using data source
404/// combinators.
405#[derive(Derivative)]
406#[derivative(Clone(bound = ""), Debug(bound = "S: Debug, P: Debug"))]
407pub struct FetchingDataSource<Types, S, P>
408where
409    Types: NodeType,
410{
411    // The fetcher manages retrieval of resources from both local storage and a remote provider. It
412    // encapsulates the data which may need to be shared with a long-lived task or future that
413    // implements the asynchronous fetching of a particular object. This is why it gets its own
414    // type, wrapped in an [`Arc`] for easy, efficient cloning.
415    fetcher: Arc<Fetcher<Types, S, P>>,
416    // The proactive scanner task. This is only saved here so that we can cancel it on drop.
417    scanner: Option<BackgroundTask>,
418    // The aggregator task, which derives aggregate statistics from a block stream.
419    aggregator: Option<BackgroundTask>,
420    pruner: Pruner<Types, S>,
421}
422
423#[derive(Derivative)]
424#[derivative(Clone(bound = ""), Debug(bound = "S: Debug,   "))]
425pub struct Pruner<Types, S>
426where
427    Types: NodeType,
428{
429    handle: Option<BackgroundTask>,
430    _types: PhantomData<(Types, S)>,
431}
432
433impl<Types, S> Pruner<Types, S>
434where
435    Types: NodeType,
436    Payload<Types>: QueryablePayload<Types>,
437    S: PruneStorage + Send + Sync + 'static,
438{
439    async fn new(storage: Arc<S>) -> Self {
440        let cfg = storage.get_pruning_config();
441        let Some(cfg) = cfg else {
442            return Self {
443                handle: None,
444                _types: Default::default(),
445            };
446        };
447
448        let future = async move {
449            for i in 1.. {
450                tracing::warn!("starting pruner run {i} ");
451                Self::prune(storage.clone()).await;
452                sleep(cfg.interval()).await;
453            }
454        };
455
456        let task = BackgroundTask::spawn("pruner", future);
457
458        Self {
459            handle: Some(task),
460            _types: Default::default(),
461        }
462    }
463
464    async fn prune(storage: Arc<S>) {
465        // We loop until the whole run pruner run is complete
466        let mut pruner = S::Pruner::default();
467        loop {
468            match storage.prune(&mut pruner).await {
469                Ok(Some(height)) => {
470                    tracing::warn!("Pruned to height {height}");
471                },
472                Ok(None) => {
473                    tracing::warn!("pruner run complete.");
474                    break;
475                },
476                Err(e) => {
477                    tracing::error!("pruner run failed: {e:?}");
478                    break;
479                },
480            }
481        }
482    }
483}

485impl<Types, S, P> FetchingDataSource<Types, S, P>
486where
487    Types: NodeType,
488    Payload<Types>: QueryablePayload<Types>,
489    Header<Types>: QueryableHeader<Types>,
490    S: VersionedDataSource + PruneStorage + HasMetrics + MigrateTypes<Types> + 'static,
491    for<'a> S::Transaction<'a>: UpdateAvailabilityStorage<Types> + UpdateAggregatesStorage<Types>,
492    for<'a> S::ReadOnly<'a>:
493        AvailabilityStorage<Types> + NodeStorage<Types> + PrunedHeightStorage + AggregatesStorage,
494    P: AvailabilityProvider<Types>,
495{
496    /// Build a [`FetchingDataSource`] with the given `storage` and `provider`.
497    pub fn builder(storage: S, provider: P) -> Builder<Types, S, P> {
498        Builder::new(storage, provider)
499    }
500
501    async fn new(builder: Builder<Types, S, P>) -> anyhow::Result<Self> {
502        let leaf_only = builder.is_leaf_only();
503        let aggregator = builder.aggregator;
504        let aggregator_chunk_size = builder
505            .aggregator_chunk_size
506            .unwrap_or(builder.range_chunk_size);
507        let proactive_fetching = builder.proactive_fetching;
508        let minor_interval = builder.minor_scan_interval;
509        let major_interval = builder.major_scan_interval;
510        let major_offset = builder.major_scan_offset;
511        let proactive_range_chunk_size = builder
512            .proactive_range_chunk_size
513            .unwrap_or(builder.range_chunk_size);
514        let scanner_metrics = ScannerMetrics::new(builder.storage.metrics());
515        let aggregator_metrics = AggregatorMetrics::new(builder.storage.metrics());
516
517        let fetcher = Arc::new(Fetcher::new(builder).await?);
518
519        // Migrate the old types to new PoS types
520        // This is a one-time operation that should be done before starting the data source
521        // It migrates leaf1 storage to leaf2
522        // and vid to vid2
523        fetcher.storage.migrate_types().await?;
524
525        let scanner = if proactive_fetching && !leaf_only {
526            Some(BackgroundTask::spawn(
527                "proactive scanner",
528                fetcher.clone().proactive_scan(
529                    minor_interval,
530                    major_interval,
531                    major_offset,
532                    proactive_range_chunk_size,
533                    scanner_metrics,
534                ),
535            ))
536        } else {
537            None
538        };
539
540        let aggregator = if aggregator && !leaf_only {
541            Some(BackgroundTask::spawn(
542                "aggregator",
543                fetcher
544                    .clone()
545                    .aggregate(aggregator_chunk_size, aggregator_metrics),
546            ))
547        } else {
548            None
549        };
550
551        let storage = fetcher.storage.clone();
552
553        let pruner = Pruner::new(storage).await;
554        let ds = Self {
555            fetcher,
556            scanner,
557            pruner,
558            aggregator,
559        };
560
561        Ok(ds)
562    }
563}

947impl<Types, S, P> Fetcher<Types, S, P>
948where
949    Types: NodeType,
950    Payload<Types>: QueryablePayload<Types>,
951    S: VersionedDataSource + 'static,
952    for<'a> S::Transaction<'a>: UpdateAvailabilityStorage<Types>,
953    for<'a> S::ReadOnly<'a>: AvailabilityStorage<Types> + NodeStorage<Types> + PrunedHeightStorage,
954    P: AvailabilityProvider<Types>,
955{
956    async fn get<T>(self: &Arc<Self>, req: impl Into<T::Request> + Send) -> Fetch<T>
957    where
958        T: Fetchable<Types>,
959    {
960        let req = req.into();
961
962        // Subscribe to notifications before we check storage for the requested object. This ensures
963        // that this operation will always eventually succeed as long as the requested object
964        // actually exists (or will exist). We will either find it in our local storage and succeed
965        // immediately, or (if it exists) someone will *later* come and add it to storage, at which
966        // point we will get a notification causing this passive fetch to resolve.
967        //
968        // Note the "someone" who later fetches the object and adds it to storage may be an active
969        // fetch triggered by this very requests, in cases where that is possible, but it need not
970        // be.
971        let passive_fetch = T::passive_fetch(&self.notifiers, req).await;
972
973        match self.try_get(req).await {
974            Ok(Some(obj)) => return Fetch::Ready(obj),
975            Ok(None) => return passive(req, passive_fetch),
976            Err(err) => {
977                tracing::warn!(
978                    ?req,
979                    "unable to fetch object; spawning a task to retry: {err:#}"
980                );
981            },
982        }
983
984        // We'll use this channel to get the object back if we successfully load it on retry.
985        let (send, recv) = oneshot::channel();
986
987        let fetcher = self.clone();
988        let mut backoff = fetcher.backoff.clone();
989        let span = tracing::warn_span!("get retry", ?req);
990        spawn(
991            async move {
992                let mut delay = backoff.next_backoff().unwrap_or(Duration::from_secs(1));
993                loop {
994                    let res = {
995                        // Limit the number of simultaneous retry tasks hitting the database. When
996                        // the database is down, we might have a lot of these tasks running, and if
997                        // they all hit the DB at once, they are only going to make things worse.
998                        let _guard = fetcher.retry_semaphore.acquire().await;
999                        fetcher.try_get(req).await
1000                    };
1001                    match res {
1002                        Ok(Some(obj)) => {
1003                            // If the object was immediately available after all, signal the
1004                            // original fetch. We probably just temporarily couldn't access it due
1005                            // to database errors.
1006                            tracing::info!(?req, "object was ready after retries");
1007                            send.send(obj).ok();
1008                            break;
1009                        },
1010                        Ok(None) => {
1011                            // The object was not immediately available after all, but we have
1012                            // successfully spawned a fetch for it if possible. The spawned fetch
1013                            // will notify the original request once it completes.
1014                            tracing::info!(?req, "spawned fetch after retries");
1015                            break;
1016                        },
1017                        Err(err) => {
1018                            tracing::warn!(
1019                                ?req,
1020                                ?delay,
1021                                "unable to fetch object, will retry: {err:#}"
1022                            );
1023                            sleep(delay).await;
1024                            if let Some(next_delay) = backoff.next_backoff() {
1025                                delay = next_delay;
1026                            }
1027                        },
1028                    }
1029                }
1030            }
1031            .instrument(span),
1032        );
1033
1034        // Wait for the object to be fetched, either from the local database on retry or from
1035        // another provider eventually.
1036        passive(req, select_some(passive_fetch, recv.map(Result::ok)))
1037    }

1039    /// Try to get an object from local storage or initialize a fetch if it is missing.
1040    ///
1041    /// There are three possible scenarios in this function, indicated by the return type:
1042    /// * `Ok(Some(obj))`: the requested object was available locally and successfully retrieved
1043    ///   from the database; no fetch was spawned
1044    /// * `Ok(None)`: the requested object was not available locally, but a fetch was successfully
1045    ///   spawned if possible (in other words, if a fetch was not spawned, it was determined that
1046    ///   the requested object is not fetchable)
1047    /// * `Err(_)`: it could not be determined whether the object was available locally or whether
1048    ///   it could be fetched; no fetch was spawned even though the object may be fetchable
1049    async fn try_get<T>(self: &Arc<Self>, req: T::Request) -> anyhow::Result<Option<T>>
1050    where
1051        T: Fetchable<Types>,
1052    {
1053        let mut tx = self.read().await.context("opening read transaction")?;
1054        match T::load(&mut tx, req).await {
1055            Ok(t) => Ok(Some(t)),
1056            Err(QueryError::Missing | QueryError::NotFound) => {
1057                // We successfully queried the database, but the object wasn't there. Try to
1058                // fetch it.
1059                tracing::debug!(?req, "object missing from local storage, will try to fetch");
1060                self.fetch::<T>(&mut tx, req).await?;
1061                Ok(None)
1062            },
1063            Err(err) => {
1064                // An error occurred while querying the database. We don't know if we need to fetch
1065                // the object or not. Return an error so we can try again.
1066                bail!("failed to fetch resource {req:?} from local storage: {err:#}");
1067            },
1068        }
1069    }

1071    /// Get a range of objects from local storage or a provider.
1072    ///
1073    /// Convert a finite stream of fallible local storage lookups into a (possibly infinite) stream
1074    /// of infallible fetches. Objects in `range` are loaded from local storage. Any gaps or missing
1075    /// objects are filled by fetching from a provider. Items in the resulting stream are futures
1076    /// that will never fail to produce a resource, although they may block indefinitely if the
1077    /// resource needs to be fetched.
1078    ///
1079    /// Objects are loaded and fetched in chunks, which strikes a good balance of limiting the total
1080    /// number of storage and network requests, while also keeping the amount of simultaneous
1081    /// resource consumption bounded.
1082    fn get_range<R, T>(self: Arc<Self>, range: R) -> BoxStream<'static, Fetch<T>>
1083    where
1084        R: RangeBounds<usize> + Send + 'static,
1085        T: RangedFetchable<Types>,
1086    {
1087        let chunk_size = self.range_chunk_size;
1088        self.get_range_with_chunk_size(chunk_size, range)
1089    }

1091    /// Same as [`Self::get_range`], but uses the given chunk size instead of the default.
1092    fn get_range_with_chunk_size<R, T>(
1093        self: Arc<Self>,
1094        chunk_size: usize,
1095        range: R,
1096    ) -> BoxStream<'static, Fetch<T>>
1097    where
1098        R: RangeBounds<usize> + Send + 'static,
1099        T: RangedFetchable<Types>,
1100    {
1101        let chunk_fetch_delay = self.chunk_fetch_delay;
1102        let active_fetch_delay = self.active_fetch_delay;
1103
1104        stream::iter(range_chunks(range, chunk_size))
1105            .then(move |chunk| {
1106                let self_clone = self.clone();
1107                async move {
1108                    {
1109                        let chunk = self_clone.get_chunk(chunk).await;
1110
1111                        // Introduce a delay (`chunk_fetch_delay`) between fetching chunks. This
1112                        // helps to limit constant high CPU usage when fetching long range of data,
1113                        // especially for older streams that fetch most of the data from local
1114                        // storage.
1115                        sleep(chunk_fetch_delay).await;
1116                        stream::iter(chunk)
1117                    }
1118                }
1119            })
1120            .flatten()
1121            .then(move |f| async move {
1122                match f {
1123                    // Introduce a delay (`active_fetch_delay`) for active fetches to reduce load on
1124                    // the catchup provider. The delay applies between pending fetches, not between
1125                    // chunks.
1126                    Fetch::Pending(_) => sleep(active_fetch_delay).await,
1127                    Fetch::Ready(_) => (),
1128                };
1129                f
1130            })
1131            .boxed()
1132    }

1195    /// Get a range of objects from local storage or a provider.
1196    ///
1197    /// This method is similar to `get_range`, except that:
1198    /// * It fetches all desired objects together, as a single chunk
1199    /// * It loads the object or triggers fetches right now rather than providing a lazy stream
1200    ///   which only fetches objects when polled.
1201    async fn get_chunk<T>(self: &Arc<Self>, chunk: Range<usize>) -> Vec<Fetch<T>>
1202    where
1203        T: RangedFetchable<Types>,
1204    {
1205        // Subscribe to notifications first. As in [`get`](Self::get), this ensures we won't miss
1206        // any notifications sent in between checking local storage and triggering a fetch if
1207        // necessary.
1208        let passive_fetches = join_all(
1209            chunk
1210                .clone()
1211                .map(|i| T::passive_fetch(&self.notifiers, i.into())),
1212        )
1213        .await;
1214
1215        match self.try_get_chunk(&chunk).await {
1216            Ok(objs) => {
1217                // Convert to fetches. Objects which are not immediately available (`None` in the
1218                // chunk) become passive fetches awaiting a notification of availability.
1219                return objs
1220                    .into_iter()
1221                    .zip(passive_fetches)
1222                    .enumerate()
1223                    .map(move |(i, (obj, passive_fetch))| match obj {
1224                        Some(obj) => Fetch::Ready(obj),
1225                        None => passive(T::Request::from(chunk.start + i), passive_fetch),
1226                    })
1227                    .collect();
1228            },
1229            Err(err) => {
1230                tracing::warn!(
1231                    ?chunk,
1232                    "unable to fetch chunk; spawning a task to retry: {err:#}"
1233                );
1234            },
1235        }
1236
1237        // We'll use these channels to get the objects back that we successfully load on retry.
1238        let (send, recv): (Vec<_>, Vec<_>) =
1239            repeat_with(oneshot::channel).take(chunk.len()).unzip();
1240
1241        {
1242            let fetcher = self.clone();
1243            let mut backoff = fetcher.backoff.clone();
1244            let chunk = chunk.clone();
1245            let span = tracing::warn_span!("get_chunk retry", ?chunk);
1246            spawn(
1247                async move {
1248                    let mut delay = backoff.next_backoff().unwrap_or(Duration::from_secs(1));
1249                    loop {
1250                        let res = {
1251                            // Limit the number of simultaneous retry tasks hitting the database.
1252                            // When the database is down, we might have a lot of these tasks
1253                            // running, and if they all hit the DB at once, they are only going to
1254                            // make things worse.
1255                            let _guard = fetcher.retry_semaphore.acquire().await;
1256                            fetcher.try_get_chunk(&chunk).await
1257                        };
1258                        match res {
1259                            Ok(objs) => {
1260                                for (i, (obj, sender)) in objs.into_iter().zip(send).enumerate() {
1261                                    if let Some(obj) = obj {
1262                                        // If the object was immediately available after all, signal
1263                                        // the original fetch. We probably just temporarily couldn't
1264                                        // access it due to database errors.
1265                                        tracing::info!(?chunk, i, "object was ready after retries");
1266                                        sender.send(obj).ok();
1267                                    } else {
1268                                        // The object was not immediately available after all, but
1269                                        // we have successfully spawned a fetch for it if possible.
1270                                        // The spawned fetch will notify the original request once
1271                                        // it completes.
1272                                        tracing::info!(?chunk, i, "spawned fetch after retries");
1273                                    }
1274                                }
1275                                break;
1276                            },
1277                            Err(err) => {
1278                                tracing::warn!(
1279                                    ?chunk,
1280                                    ?delay,
1281                                    "unable to fetch chunk, will retry: {err:#}"
1282                                );
1283                                sleep(delay).await;
1284                                if let Some(next_delay) = backoff.next_backoff() {
1285                                    delay = next_delay;
1286                                }
1287                            },
1288                        }
1289                    }
1290                }
1291                .instrument(span),
1292            );
1293        }
1294
1295        // Wait for the objects to be fetched, either from the local database on retry or from
1296        // another provider eventually.
1297        passive_fetches
1298            .into_iter()
1299            .zip(recv)
1300            .enumerate()
1301            .map(move |(i, (passive_fetch, recv))| {
1302                passive(
1303                    T::Request::from(chunk.start + i),
1304                    select_some(passive_fetch, recv.map(Result::ok)),
1305                )
1306            })
1307            .collect()
1308    }
1309
1310    /// Try to get a range of objects from local storage, initializing fetches if any are missing.
1311    ///
1312    /// If this function succeeded, then for each object in the requested range, either:
1313    /// * the object was available locally, and corresponds to `Some(_)` object in the result
1314    /// * the object was not available locally (and corresponds to `None` in the result), but a
1315    ///   fetch was successfully spawned if possible (in other words, if a fetch was not spawned, it
1316    ///   was determined that the requested object is not fetchable)
1317    ///
1318    /// This function will fail if it could not be determined which objects in the requested range
1319    /// are available locally, or if, for any missing object, it could not be determined whether
1320    /// that object is fetchable. In this case, there may be no fetch spawned for certain objects in
1321    /// the requested range, even if those objects are actually fetchable.
1322    async fn try_get_chunk<T>(
1323        self: &Arc<Self>,
1324        chunk: &Range<usize>,
1325    ) -> anyhow::Result<Vec<Option<T>>>
1326    where
1327        T: RangedFetchable<Types>,
1328    {
1329        let mut tx = self.read().await.context("opening read transaction")?;
1330        let ts = T::load_range(&mut tx, chunk.clone())
1331            .await
1332            .context(format!("when fetching items in range {chunk:?}"))?;
1333
1334        // Log and discard error information; we want a list of Option where None indicates an
1335        // object that needs to be fetched. Note that we don't use `FetchRequest::might_exist` to
1336        // silence the logs here when an object is missing that is not expected to exist at all.
1337        // When objects are not expected to exist, `load_range` should just return a truncated list
1338        // rather than returning `Err` objects, so if there are errors in here they are unexpected
1339        // and we do want to log them.
1340        let ts = ts.into_iter().filter_map(ResultExt::ok_or_trace);
1341
1342        // Kick off a fetch for each missing object.
1343        let mut results = Vec::with_capacity(chunk.len());
1344        for t in ts {
1345            // Fetch missing objects that should come before `t`.
1346            while chunk.start + results.len() < t.height() as usize {
1347                tracing::debug!(
1348                    "item {} in chunk not available, will be fetched",
1349                    results.len()
1350                );
1351                self.fetch::<T>(&mut tx, (chunk.start + results.len()).into())
1352                    .await?;
1353                results.push(None);
1354            }
1355
1356            results.push(Some(t));
1357        }
1358        // Fetch missing objects from the end of the range.
1359        while results.len() < chunk.len() {
1360            self.fetch::<T>(&mut tx, (chunk.start + results.len()).into())
1361                .await?;
1362            results.push(None);
1363        }
1364
1365        Ok(results)
1366    }
1367
1368    /// Spawn an active fetch for the requested object, if possible.
1369    ///
1370    /// On success, either an active fetch for `req` has been spawned, or it has been determined
1371    /// that `req` is not fetchable. Fails if it cannot be determined (e.g. due to errors in the
1372    /// local database) whether `req` is fetchable or not.
1373    async fn fetch<T>(
1374        self: &Arc<Self>,
1375        tx: &mut <Self as VersionedDataSource>::ReadOnly<'_>,
1376        req: T::Request,
1377    ) -> anyhow::Result<()>
1378    where
1379        T: Fetchable<Types>,
1380    {
1381        tracing::debug!("fetching resource {req:?}");
1382
1383        // Trigger an active fetch from a remote provider if possible.
1384        let heights = Heights::load(tx)
1385            .await
1386            .context("failed to load heights; cannot definitively say object might exist")?;
1387        if req.might_exist(heights) {
1388            T::active_fetch(tx, self.clone(), req).await?;
1389        } else {
1390            tracing::debug!("not fetching object {req:?} that cannot exist at {heights:?}");
1391        }
1392        Ok(())
1393    }
1394
1395    /// Proactively search for and retrieve missing objects.
1396    ///
1397    /// This function will proactively identify and retrieve blocks and leaves which are missing
1398    /// from storage. It will run until cancelled, thus, it is meant to be spawned as a background
1399    /// task rather than called synchronously.
1400    async fn proactive_scan(
1401        self: Arc<Self>,
1402        minor_interval: Duration,
1403        major_interval: usize,
1404        major_offset: usize,
1405        chunk_size: usize,
1406        metrics: ScannerMetrics,
1407    ) {
1408        let mut prev_height = 0;
1409
1410        for i in 0.. {
1411            let major = i % major_interval == major_offset % major_interval;
1412            let span = tracing::warn_span!("proactive scan", i, major, prev_height);
1413            metrics.running.set(1);
1414            metrics.current_scan.set(i);
1415            metrics.current_is_major.set(major as usize);
1416            async {
1417                let mut backoff = minor_interval;
1418                let max_backoff = Duration::from_secs(60);
1419                metrics.backoff.set(backoff.as_secs() as usize);
1420
1421                // We can't start the scan until we know the current block height and pruned height,
1422                // so we know which blocks to scan. Thus we retry until this succeeds.
1423                let heights = loop {
1424                    let mut tx = match self.read().await {
1425                        Ok(tx) => tx,
1426                        Err(err) => {
1427                            tracing::error!(
1428                                ?backoff,
1429                                "unable to start transaction for scan: {err:#}"
1430                            );
1431                            metrics.retries.update(1);
1432                            sleep(backoff).await;
1433                            backoff = min(2 * backoff, max_backoff);
1434                            metrics.backoff.set(backoff.as_secs() as usize);
1435                            continue;
1436                        },
1437                    };
1438                    let heights = match Heights::load(&mut tx).await {
1439                        Ok(heights) => heights,
1440                        Err(err) => {
1441                            tracing::error!(?backoff, "unable to load heights: {err:#}");
1442                            metrics.retries.update(1);
1443                            sleep(backoff).await;
1444                            backoff = min(2 * backoff, max_backoff);
1445                            metrics.backoff.set(backoff.as_secs() as usize);
1446                            continue;
1447                        },
1448                    };
1449                    metrics.retries.set(0);
1450                    break heights;
1451                };
1452
1453                // Get the pruned height or default to 0 if it is not set. We will not look for
1454                // blocks older than the pruned height.
1455                let minimum_block_height = heights.pruned_height.unwrap_or(0) as usize;
1456                // Get the block height; we will look for any missing blocks up to `block_height`.
1457                let block_height = heights.height as usize;
1458
1459                // In a major scan, we fetch all blocks between 0 and `block_height`. In minor scans
1460                // (much more frequent) we fetch blocks that are missing since the last scan.
1461                let start = if major {
1462                    // We log major scans at WARN level, since they happen infrequently and can have
1463                    // a measurable impact on performance while running. This also serves as a
1464                    // useful progress heartbeat.
1465                    tracing::warn!(
1466                        start = minimum_block_height,
1467                        block_height,
1468                        "starting major scan"
1469                    );
1470
1471                    // If we're starting a major scan, reset the major scan counts of missing data
1472                    // as we're about to recompute them.
1473                    metrics.major_missing_blocks.set(0);
1474                    metrics.major_missing_vid.set(0);
1475
1476                    minimum_block_height
1477                } else {
1478                    tracing::info!(start = prev_height, block_height, "starting minor scan");
1479                    prev_height
1480                };
1481                prev_height = block_height;
1482                metrics.current_start.set(start);
1483                metrics.current_end.set(block_height);
1484                metrics.scanned_blocks.set(0);
1485                metrics.scanned_vid.set(0);
1486
1487                // Iterate over all blocks that we should have. Fetching the payload metadata is
1488                // enough to trigger an active fetch of the corresponding leaf and the full block if
1489                // they are missing, without loading the full payload from storage if we already
1490                // have it.
1491                //
1492                // We iterate in reverse order for two reasons:
1493                // 1. More recent blocks are more likely to be requested sooner.
1494                // 2. Leaves are inherently fetched in reverse order, because we cannot (actively)
1495                //    fetch a leaf until we have the subsequent leaf, which tells us what the hash
1496                //    of its parent should be. Thus, if we iterated forwards, we could get stuck any
1497                //    time we came upon two consecutive missing leaves.
1498                let mut blocks = self
1499                    .clone()
1500                    .get_range_with_chunk_size_rev::<PayloadMetadata<Types>>(
1501                        chunk_size,
1502                        Bound::Included(start),
1503                        block_height.saturating_sub(1),
1504                    );
1505                let mut missing_blocks = 0;
1506                while let Some(fetch) = blocks.next().await {
1507                    if fetch.is_pending() {
1508                        missing_blocks += 1;
1509                        // Wait for the block to be fetched. This slows down the scanner so that we
1510                        // don't waste memory generating more active fetch tasks then we can handle
1511                        // at a given time. Note that even with this await, all blocks within a
1512                        // chunk are fetched in parallel, so this does not block the next block in
1513                        // the chunk, only the next chunk until the current chunk completes.
1514                        fetch.await;
1515                    }
1516                    metrics.scanned_blocks.update(1);
1517                }
1518                metrics.add_missing_blocks(major, missing_blocks);
1519
1520                // We have to trigger a separate fetch of the VID data, since this is fetched
1521                // independently of the block payload.
1522                let mut vid = self
1523                    .clone()
1524                    .get_range_with_chunk_size_rev::<VidCommonMetadata<Types>>(
1525                        chunk_size,
1526                        Bound::Included(start),
1527                        block_height.saturating_sub(1),
1528                    );
1529                let mut missing_vid = 0;
1530                while let Some(fetch) = vid.next().await {
1531                    if fetch.is_pending() {
1532                        missing_vid += 1;
1533                        // As above, limit the speed at which we spawn new fetches to the speed at
1534                        // which we can process them.
1535                        fetch.await;
1536                    }
1537                    metrics.scanned_vid.update(1);
1538                }
1539                metrics.add_missing_vid(major, missing_vid);
1540
1541                tracing::info!("completed proactive scan, will scan again in {minor_interval:?}");
1542
1543                // Reset metrics.
1544                metrics.running.set(0);
1545                if major {
1546                    // If we just completed a major scan, reset the incremental counts of missing
1547                    // data from minor scans, so the next round of minor scans can recompute/update
1548                    // them.
1549                    metrics.minor_missing_blocks.set(0);
1550                    metrics.minor_missing_vid.set(0);
1551                }
1552
1553                sleep(minor_interval).await;
1554            }
1555            .instrument(span)
1556            .await;
1557        }
1558    }
1559}

1561impl<Types, S, P> Fetcher<Types, S, P>
1562where
1563    Types: NodeType,
1564    Payload<Types>: QueryablePayload<Types>,
1565    S: VersionedDataSource + 'static,
1566    for<'a> S::Transaction<'a>: UpdateAvailabilityStorage<Types> + UpdateAggregatesStorage<Types>,
1567    for<'a> S::ReadOnly<'a>:
1568        AvailabilityStorage<Types> + NodeStorage<Types> + PrunedHeightStorage + AggregatesStorage,
1569    P: AvailabilityProvider<Types>,
1570{
1571    #[tracing::instrument(skip_all)]
1572    async fn aggregate(self: Arc<Self>, chunk_size: usize, metrics: AggregatorMetrics) {
1573        loop {
1574            let prev_aggregate = loop {
1575                let mut tx = match self.read().await {
1576                    Ok(tx) => tx,
1577                    Err(err) => {
1578                        tracing::error!("unable to open read tx: {err:#}");
1579                        sleep(Duration::from_secs(5)).await;
1580                        continue;
1581                    },
1582                };
1583                match tx.load_prev_aggregate().await {
1584                    Ok(agg) => break agg,
1585                    Err(err) => {
1586                        tracing::error!("unable to load previous aggregate: {err:#}");
1587                        sleep(Duration::from_secs(5)).await;
1588                        continue;
1589                    },
1590                }
1591            };
1592
1593            let (start, mut prev_aggregate) = match prev_aggregate {
1594                Some(aggregate) => (aggregate.height as usize + 1, aggregate),
1595                None => (0, Aggregate::default()),
1596            };
1597
1598            tracing::info!(start, "starting aggregator");
1599            metrics.height.set(start);
1600
1601            let mut blocks = self
1602                .clone()
1603                .get_range_with_chunk_size::<_, PayloadMetadata<Types>>(chunk_size, start..)
1604                .then(Fetch::resolve)
1605                .ready_chunks(chunk_size)
1606                .boxed();
1607            while let Some(chunk) = blocks.next().await {
1608                let Some(last) = chunk.last() else {
1609                    // This is not supposed to happen, but if the chunk is empty, just skip it.
1610                    tracing::warn!("ready_chunks returned an empty chunk");
1611                    continue;
1612                };
1613                let height = last.height();
1614                let num_blocks = chunk.len();
1615                tracing::debug!(
1616                    num_blocks,
1617                    height,
1618                    "updating aggregate statistics for chunk"
1619                );
1620                loop {
1621                    let res = async {
1622                        let mut tx = self.write().await.context("opening transaction")?;
1623                        let aggregate =
1624                            tx.update_aggregates(prev_aggregate.clone(), &chunk).await?;
1625                        tx.commit().await.context("committing transaction")?;
1626                        prev_aggregate = aggregate;
1627                        anyhow::Result::<_>::Ok(())
1628                    }
1629                    .await;
1630                    match res {
1631                        Ok(()) => {
1632                            break;
1633                        },
1634                        Err(err) => {
1635                            tracing::warn!(
1636                                num_blocks,
1637                                height,
1638                                "failed to update aggregates for chunk: {err:#}"
1639                            );
1640                            sleep(Duration::from_secs(1)).await;
1641                        },
1642                    }
1643                }
1644                metrics.height.set(height as usize);
1645            }
1646            tracing::warn!("aggregator block stream ended unexpectedly; will restart");
1647        }
1648    }
1649}

1651impl<Types, S, P> Fetcher<Types, S, P>
1652where
1653    Types: NodeType,
1654    S: VersionedDataSource,
1655    for<'a> S::Transaction<'a>: UpdateAvailabilityStorage<Types>,
1656{
1657    /// Store an object and notify anyone waiting on this object that it is available.
1658    async fn store_and_notify<T>(&self, obj: T)
1659    where
1660        T: Storable<Types>,
1661    {
1662        let try_store = || async {
1663            let mut tx = self.storage.write().await?;
1664            obj.clone().store(&mut tx, self.leaf_only).await?;
1665            tx.commit().await
1666        };
1667
1668        // Store the object in local storage, so we can avoid fetching it in the future.
1669        let mut backoff = self.backoff.clone();
1670        backoff.reset();
1671        loop {
1672            let Err(err) = try_store().await else {
1673                break;
1674            };
1675            // It is unfortunate if this fails, but we can still proceed by notifying with the
1676            // object that we fetched, keeping it in memory. Log the error, retry a few times, and
1677            // eventually move on.
1678            tracing::warn!(
1679                "failed to store fetched {} {}: {err:#}",
1680                T::name(),
1681                obj.height()
1682            );
1683
1684            let Some(delay) = backoff.next_backoff() else {
1685                break;
1686            };
1687            tracing::info!(?delay, "retrying failed operation");
1688            sleep(delay).await;
1689        }
1690
1691        // Send a notification about the newly received object. It is important that we do this
1692        // _after_ our attempt to store the object in local storage, otherwise there is a potential
1693        // missed notification deadlock:
1694        // * we send the notification
1695        // * a task calls [`get`](Self::get) or [`get_chunk`](Self::get_chunk), finds that the
1696        //   requested object is not in storage, and begins waiting for a notification
1697        // * we store the object. This ensures that no other task will be triggered to fetch it,
1698        //   which means no one will ever notify the waiting task.
1699        //
1700        // Note that we send the notification regardless of whether the store actually succeeded or
1701        // not. This is to avoid _another_ subtle deadlock: if we failed to notify just because we
1702        // failed to store, some fetches might not resolve, even though the object in question has
1703        // actually been fetched. This should actually be ok, because as long as the object is not
1704        // in storage, eventually some other task will come along and fetch, store, and notify about
1705        // it. However, this is certainly not ideal, since we could resolve those pending fetches
1706        // right now, and it causes bigger problems when the fetch that fails to resolve is the
1707        // proactive scanner task, who is often the one that would eventually come along and
1708        // re-fetch the object.
1709        //
1710        // The key thing to note is that it does no harm to notify even if we fail to store: at best
1711        // we wake some tasks up sooner; at worst, anyone who misses the notification still
1712        // satisfies the invariant that we only wait on notifications for objects which are not in
1713        // storage, and eventually some other task will come along, find the object missing from
1714        // storage, and re-fetch it.
1715        obj.notify(&self.notifiers).await;
1716    }
1717}

1996/// Objects which can be fetched from a remote DA provider and cached in local storage.
1997///
1998/// This trait lets us abstract over leaves, blocks, and other types that can be fetched. Thus, the
1999/// logistics of fetching are shared between all objects, and only the low-level particulars are
2000/// type-specific.
2001#[async_trait]
2002trait Fetchable<Types>: Clone + Send + Sync + 'static
2003where
2004    Types: NodeType,
2005    Payload<Types>: QueryablePayload<Types>,
2006{
2007    /// A succinct specification of the object to be fetched.
2008    type Request: FetchRequest;
2009
2010    /// Does this object satisfy the given request?
2011    fn satisfies(&self, req: Self::Request) -> bool;
2012
2013    /// Spawn a task to fetch the object from a remote provider, if possible.
2014    ///
2015    /// An active fetch will only be triggered if:
2016    /// * There is not already an active fetch in progress for the same object
2017    /// * The requested object is known to exist. For example, we will fetch a leaf by height but
2018    ///   not by hash, since we can't guarantee that a leaf with an arbitrary hash exists. Note that
2019    ///   this function assumes `req.might_exist()` has already been checked before calling it, and
2020    ///   so may do unnecessary work if the caller does not ensure this.
2021    ///
2022    /// If we do trigger an active fetch for an object, any passive listeners for the object will be
2023    /// notified once it has been retrieved. If we do not trigger an active fetch for an object,
2024    /// this function does nothing. In either case, as long as the requested object does in fact
2025    /// exist, we will eventually receive it passively, since we will eventually receive all blocks
2026    /// and leaves that are ever produced. Active fetching merely helps us receive certain objects
2027    /// sooner.
2028    ///
2029    /// This function fails if it _might_ be possible to actively fetch the requested object, but we
2030    /// were unable to do so (e.g. due to errors in the database).
2031    async fn active_fetch<S, P>(
2032        tx: &mut impl AvailabilityStorage<Types>,
2033        fetcher: Arc<Fetcher<Types, S, P>>,
2034        req: Self::Request,
2035    ) -> anyhow::Result<()>
2036    where
2037        S: VersionedDataSource + 'static,
2038        for<'a> S::Transaction<'a>: UpdateAvailabilityStorage<Types>,
2039        for<'a> S::ReadOnly<'a>:
2040            AvailabilityStorage<Types> + NodeStorage<Types> + PrunedHeightStorage,
2041        P: AvailabilityProvider<Types>;
2042
2043    /// Wait for someone else to fetch the object.
2044    async fn passive_fetch(notifiers: &Notifiers<Types>, req: Self::Request) -> PassiveFetch<Self>;
2045
2046    /// Load an object from local storage.
2047    ///
2048    /// This function assumes `req.might_exist()` has already been checked before calling it, and so
2049    /// may do unnecessary work if the caller does not ensure this.
2050    async fn load<S>(storage: &mut S, req: Self::Request) -> QueryResult<Self>
2051    where
2052        S: AvailabilityStorage<Types>;
2053}

2123/// Break a range into fixed-size chunks.
2124fn range_chunks<R>(range: R, chunk_size: usize) -> impl Iterator<Item = Range<usize>>
2125where
2126    R: RangeBounds<usize>,
2127{
2128    // Transform range to explicit start (inclusive) and end (exclusive) bounds.
2129    let mut start = match range.start_bound() {
2130        Bound::Included(i) => *i,
2131        Bound::Excluded(i) => *i + 1,
2132        Bound::Unbounded => 0,
2133    };
2134    let end = match range.end_bound() {
2135        Bound::Included(i) => *i + 1,
2136        Bound::Excluded(i) => *i,
2137        Bound::Unbounded => usize::MAX,
2138    };
2139    std::iter::from_fn(move || {
2140        let chunk_end = min(start + chunk_size, end);
2141        if chunk_end == start {
2142            return None;
2143        }
2144
2145        let chunk = start..chunk_end;
2146        start = chunk_end;
2147        Some(chunk)
2148    })
2149}
2150
2151/// Break a range into fixed-size chunks, starting from the end and moving towards the start.
2152///
2153/// While the chunks are yielded in reverse order, from `end` to `start`, each individual chunk is
2154/// in the usual ascending order. That is, the first chunk ends with `end` and the last chunk starts
2155/// with `start`.
2156///
2157/// Note that unlike [`range_chunks_rev`], which accepts any range and yields an infinite iterator
2158/// if the range has no upper bound, this function requires there to be a defined upper bound,
2159/// otherwise we don't know where the reversed iterator should _start_. The `end` bound given here
2160/// is inclusive; i.e. the end of the first chunk yielded by the stream will be exactly `end`.
2161fn range_chunks_rev(
2162    start: Bound<usize>,
2163    end: usize,
2164    chunk_size: usize,
2165) -> impl Iterator<Item = Range<usize>> {
2166    // Transform the start bound to be inclusive.
2167    let start = match start {
2168        Bound::Included(i) => i,
2169        Bound::Excluded(i) => i + 1,
2170        Bound::Unbounded => 0,
2171    };
2172    // Transform the end bound to be exclusive.
2173    let mut end = end + 1;
2174
2175    std::iter::from_fn(move || {
2176        let chunk_start = max(start, end.saturating_sub(chunk_size));
2177        if end <= chunk_start {
2178            return None;
2179        }
2180
2181        let chunk = chunk_start..end;
2182        end = chunk_start;
2183        Some(chunk)
2184    })
2185}

2294/// Turn a fallible passive fetch future into an infallible "fetch".
2295///
2296/// Basically, we ignore failures due to a channel sender being dropped, which should never happen.
2297fn passive<T>(
2298    req: impl Debug + Send + 'static,
2299    fut: impl Future<Output = Option<T>> + Send + 'static,
2300) -> Fetch<T>
2301where
2302    T: Send + 'static,
2303{
2304    Fetch::Pending(
2305        fut.then(move |opt| async move {
2306            match opt {
2307                Some(t) => t,
2308                None => {
2309                    // If `passive_fetch` returns `None`, it means the notifier was dropped without
2310                    // ever sending a notification. In this case, the correct behavior is actually
2311                    // to block forever (unless the `Fetch` itself is dropped), since the semantics
2312                    // of `Fetch` are to never fail. This is analogous to fetching an object which
2313                    // doesn't actually exist: the `Fetch` will never return.
2314                    //
2315                    // However, for ease of debugging, and since this is never expected to happen in
2316                    // normal usage, we panic instead. This should only happen in two cases:
2317                    // * The server was shut down (dropping the notifier) without cleaning up some
2318                    //   background tasks. This will not affect runtime behavior, but should be
2319                    //   fixed if it happens.
2320                    // * There is a very unexpected runtime bug resulting in the notifier being
2321                    //   dropped. If this happens, things are very broken in any case, and it is
2322                    //   better to panic loudly than simply block forever.
2323                    panic!("notifier dropped without satisfying request {req:?}");
2324                },
2325            }
2326        })
2327        .boxed(),
2328    )
2329}

2331/// Get the result of the first future to return `Some`, if either do.
2332async fn select_some<T>(
2333    a: impl Future<Output = Option<T>> + Unpin,
2334    b: impl Future<Output = Option<T>> + Unpin,
2335) -> Option<T> {
2336    match future::select(a, b).await {
2337        // If the first future resolves with `Some`, immediately return the result.
2338        Either::Left((Some(a), _)) => Some(a),
2339        Either::Right((Some(b), _)) => Some(b),
2340
2341        // If the first future resolves with `None`, wait for the result of the second future.
2342        Either::Left((None, b)) => b.await,
2343        Either::Right((None, a)) => a.await,
2344    }
2345}
