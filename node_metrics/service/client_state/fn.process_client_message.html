process_client_message is a function that processes the client message and processes the message accordingly.

The DataState is provided and is used only as a Read lock to distribute the current state of the system to the clients upon request.

The ClientThreadState is provided as it needs to be updated with new subscriptions / new connections depending on the incoming InternalClientMessage
